# Детальное описание проекта

## Стек технологий

- **Язык:** JavaScript / Node.js (v20)
- **Фреймворк:** Playwright Test
- **CI/CD:** GitHub Actions
- **Визуальное тестирование:** `pixelmatch` + `pngjs` (для попиксельного сравнения)
- **Работа с данными:** `exceljs` (для парсинга Excel-таблиц миграции)

## Установка и настройка (Локально)

1. Убедитесь, что установлен Node.js версии 20. ( `node -v` )
2. Склонируйте репозиторий и перейдите в ветку `aqa/parameterized`.
3. Установите зависимости:
   1. `git clone https://github.com/AnnaMark90/king-of-span-aqa.git`
   2. `cd king-of-span-aqa`
   3. `npm ci`
   4. `npx playwright install --with-deps`

## Структура проекта

- **`tests/`** — автотесты, разбитые по логическим блокам:
  - `migrationCompare/` — тесты для сравнения сред .
  - `smoke/` — п`роверки критического функционала.
  - `stageHealth/` — проверки работы ссылок на Stage.
- **`.github/workflows/`** — конфигурация CI/CD (пайплайн `playwright.yml` для ручного запуска в облаке с разбиением на 6 потоков).
- **`constants/`** — глобальные переменные и константы проекта.
- **`dataBatches/`** — наборы данных для параметризации.
- **`helpers/`** — модули с переиспользуемой логикой.
- **`pages/`** — реализация паттерна Page Object Model (`BasePage.js`, `AnyPage.js`) для хранения локаторов и методов работы со страницами.
- **`scripts/`** — вспомогательные утилиты до начала тестов (`fetchSitemap.js` для парсинга актуальных URL, `generateExcelLinks.js` для обработки Excel-таблиц).
- **`testDataExcel/`** — директория для хранения исходных таблиц миграции (`migrationTable.xlsx`).
- **`snapshots/`** — база эталонных скриншотов для визуального регресса.
  Структурирована по локалям (например, `cz-cs`, `ee-et`), брейкпоинтам (`768`, `1920`, `iPhone14`) и содержит снимки продакшена, стейджинга и файлы отличий (diffs).
- **`utils/`** — общие технические функции.
- **`playwright.config.js`** — главный файл настроек Playwright Test.
- **`package.json`** — список NPM-зависимостей и шорткаты команд для локального запуска (`npm run test`, `npm run get-urls`).

### Запуск тестов (Batch Testing)

Механизм тестирования основан на чтении файлов `.txt` из директории `dataBatches/`. Каждый файл содержит список относительных URL-адресов и представляет собой отдельный набор данных (батч). Это кастомный скрипт, парсит env.BATCH.

#### Основная команда запуска

`npm run test --batch=имя_файла`

Параметр `--batch` передает имя файла напрямую в фреймворк. Если параметр не указан, система автоматически использует `testRun.txt`.

#### NPM Скрипты в package.json (команды для запуска тестов локально)

- `npm run test` — запуск всех тестов по списку из (`testRun.txt` по умолчанию).
- `npm run test:smoke --batch=имя_файла` — запуск только быстрых смоук-проверок по выбранному списку.
- `npm run get-urls <URL_SITEMAP> <BATCH_NAME>` — автоматическое скачивание sitemap.xml, фильтрация лишних локалей и создание батча в `dataBatches/`. Наименование файла соответствует локали (регион и язык).
- `npm run get-urls:excel` — автоматический парсинг Excel-таблицы (`testDataExcel/migrationTable.xlsx`) и создание батчей для всех вкладок сразу.
- `npm run get-urls:excel -- <ИМЯ_ВКЛАДКИ>` — парсинг таблицы только для указанной локали (например, `npm run get-urls:excel -- PT-PT`).

#### Алгоритм работы с новыми локалями через Sitemap.xml

1. **Собрать данные:** Выполнить `npm run get-urls https://www.kingspan.com/pl/pl/sitemap.xml pl_pl`.
   - Скрипт сам создаст файл `dataBatches/pl_pl.txt` с очищенными ссылками.
2. **Запустить тесты:** Выполнить `npm run test --batch=pl_pl`.
3. **Проверить отчет:** После завершения выполнить `npx allure serve allure-results`.

#### Алгоритм работы с таблицей миграции (Excel)

Механизм автоматической фильтрации "битых" и цветных (красных) ячеек из таблицы миграции.

1. **Подготовить файл:** Скачать актуальную таблицу в формате `.xlsx` и положить её по пути `testDataExcel/migrationTable.xlsx`.
2. **Сгенерировать данные:** Выполнить команду `npm run get-urls:excel` (для всех) или `npm run get-urls:excel -- PT-PT` (для одной вкладки). Скрипт создаст чистые файлы в `dataBatches/`.
3. **Запустить тесты:** Выполнить `npm run test --batch=PT-PT` (для Windows PowerShell: `$env:BATCH="PT-PT"; npm run test`).
4. **Проверить отчет:** После завершения выполнить `npx playwright show-report`.

#### Приоритет выбора источника данных

Логика в `constants.js` ищет файл в следующем порядке:

1. Аргумент `--batch` из терминала.
2. Переменная окружения `BATCH`.
3. Значение по умолчанию `testRun`..

## Команды для скачивания архива Allure (CI/CD)

Скачанные файлы автоматически помещаются в изолированную директорию `allure-report/`

- **Скачать отчет последнего запуска:**
  `gh run download --name allure-html-report --dir allure-report`

- **Скачать отчет конкретного прогона (по ID):**
  `gh run download --run-id <НОМЕР_ПРОГОНА> --name allure-html-report --dir allure-report`

  _(Где `<НОМЕР_ПРОГОНА>` — это уникальный идентификатор запуска, который можно скопировать из URL пайплайна в браузере, например: `.../runs/1234567890`)._

### Просмотр скачанного отчета

Убедитесь, что вы находитесь в корне проекта, и выполните команду: `npx allure open allure-report`
